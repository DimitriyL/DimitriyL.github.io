Graph Databases

By Dimitriy Leksanov

What is a graph database?

Uses graph structure to store data, in the form of nodes, edges, and proper\ ties
Underlying structures vary between databases
Much like relational databases, graph databases rely on a query language (e\ .g. Cypher) to look up information

Why use them at all?

Significantly more efficient querying process
Links and properties thereof are stored logically within the database

Graph Database Basics

Data is stored in three major types of containers: nodes, edges, and proper\ ties

Nodes

Nodes represent “things” or “entities,” such as people, actors, movies, etc\ .
Used to store all information about the entity, much like a row of data in \ SQLite
Represented by colorful balls

Edges

Edges represent relationships between nodes
The direction of an edge denotes how nodes relate to one another
There can be multiple edges between nodes

Properties

Properties are key-value pairs that encompass the information or attributes\ assigned to nodes and edges

In nodes: these can be basic characteristics
In edges: these describe the nature of the relationship (e.g. Uber ratings)\

Different Graph Databases and Query Languages

Notable databases: GraphDB, ArangoDB, Neo4j, DataStax
Notable query languages: Cypher, Gremlin, SPARQL
These languages are standardized among numerous databases, and are imperati\ ve for data lookup


Introduction to Neo4j

Popular graph database management system
Uses the Cypher query language to add, retrieve, and remove data points
Implemented using Java
Meets the standards of ACID (Atomicity, Consistency, Isolation, Durability)\
From the official neo4j website, neo4j.com:

Atomic: All operations in a transaction succeed or every operation is rol\ led back.
Consistent: On the completion of a transaction, the database is structura\ lly sound.
Isolated: Transactions do not contend with one another. Contentious acces\ s to data is moderated by the database so that transactions appear to run seque\ ntially.
Durable: The results of applying a transaction are permanent, even in the\ presence of failures.

Basics of Cypher

For basic use of Cypher and Neo4j, there are six main key words to know: CREATE, MATCH, RETURN, WHERE, DETACH, and DELETE

CREATEing Nodes and Assigning Properties

Use the keyword CREATE, and place all information within the ensuing parentheses ()
All attributes will go inside the {}, much like a dictionary

CREATE (LanaW:Person {name:'Lana Wachowski', born:1965})
CREATE (TheMatrixReloaded:Movie {title:'The Matrix Reloaded', released:2003, tagline:'Free your mind'})

The first key-value pair represents the Node’s label, formatted as (Label_Identity):(Label_Category)

Labeling

Essentially, labels are categories for nodes
A node’s label will determine the color of the node
Much easier to distinguish in visual form
Ex: Movie→ Green; Actor → Blue

CREATEing Edges Between Nodes

Relationships are formed by larger code blocks:

CREATE
 (Keanu)-[:ACTED_IN {roles:['Neo']}]->(TheMatrix),
(Carrie)-[:ACTED_IN {roles:['Trinity']}]->(TheMatrix),
(LillyW)-[:DIRECTED]->(TheMatrix)
Nodes go within the parentheses ()
Relationships themselves go within the brackets []
Relationship labels: begin with : and are visible; properties go inside the {} and are not visible
-> used to denote the direction of the edge

Querying for Data

Use MATCH to find a node via a single property

MATCH (tom {name: "Tom Hanks"}) RETURN tom

Use WHILE to narrow down a query:

MATCH (nineties:Movie) WHERE nineties.released >= 1990 AND nineties.released <= 2000 RETURN nineties.title



“tom” and “nineties” are placeholder variables, either for a node’s label (“tom”) or a collection of nodes (“nineties”)
Here, the criterion is that each Movie in the "nineties" collection must have been "released" in the 1990s RETURN will print either a property, or the full node, as required
In this case, RETURN will print the list the titles of all movies released in the 1990s

More Advanced Searching

To shorten a list of results to a specific quantity, use LIMIT:

MATCH (nineties:Movie) WHERE nineties.released >= 1990 AND nineties.released <= 2000 RETURN nineties.title LIMIT 10

Here, the list of titles displayed will only be 10 items long, at most

Finish by Removing Connections

To clear the database, first remove edges and the relationships they correspond to using DETACH
Then, remove nodes using DELETE
These can be consolidated into one command:

MATCH (n) DETACH DELETE n

Implementing Cypher Through Drivers

In order to create and update graph databases, a developed can use drivers
In Neo4j, there are drivers for Java, JavaScript, Python, and .NET
To begin, install the driver (for Python, use pip):

$ pip install neo4j-driver

Then, running the Python driver is very similar to using Python for SQLite…

from flask import Flask, g, Response, request
from neo4j.v1 import GraphDatabase, basic_auth

with self.__driver.session() as session:
